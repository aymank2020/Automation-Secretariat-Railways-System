## 🚀 خطوات رفع المشروع على GitHub

### المشكلة الحالية:
المشروع محفوظ محلياً على جهازك لكن محتاج **GitHub Access Token** للرفع.

### الحل (خيارين):

#### الخيار 1: باستخدام Personal Access Token (الأسهل) 🔑

1. **افتح هذا الرابط:**
   👉 https://github.com/settings/tokens

2. **اضغط:**
   - Generate new token → Generate new token (classic)

3. **حدد الصلاحيات:**
   - ✅ `repo` (اختر Full control of private repositories)
   - ✅ `workflow` (اختياري)

4. **اضغط Generate token وانسخ التوكن**

5. **افتح Terminal واكتب:**
```bash
cd Automation-Secretariat-Railways-System

# استبدل YOUR_TOKEN بالتوكن اللي نسخته
git remote set-url origin https://YOUR_TOKEN@github.com/aymanbayoumi080-oss/Automation-Secretariat-Railways-System.git

# ارفع المشروع
git push -u origin main
```

#### الخيار 2: باستخدام SSH Key (أكثر أماناً) 🔐

1. **تأكيد أن عندك SSH Key:**
```bash
cat ~/.ssh/id_rsa.pub
```

لو ما ظهرت حاجة:
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
cat ~/.ssh/id_rsa.pub
```

2. **أضف الـ Public Key لـ GitHub:**
   - افتح: https://github.com/settings/ssh
   - اضبط New SSH key
   - الصق محتوى `id_rsa.pub`
   - Add SSH key

3. **رفع المشروع:**
```bash
cd Automation-Secretariat-Railways-System
git remote set-url origin git@github.com:aymanbayoumi080-oss/Automation-Secretariat-Railways-System.git
git push -u origin main
```

## 💻 كيفية تشغيل النظام على جهازك

### الطريقة 1: Docker (موصى به) 🐳

**أمر واحد فقط:**
```bash
cd Automation-Secretariat-Railways-System
./start.sh
```

**أو يدوياً:**
```bash
cd Automation-Secretariat-Railways-System

# 1. إعداد ملف البيئة
cp backend/.env.example backend/.env

# 2. تشغيل الحاويات
docker-compose up -d

# 3. إنشاء قاعدة البيانات
docker-compose exec backend python seed_db.py
```

**افتح المتصفح:**
- 🌐 http://localhost:5173
- 📖 API Docs: http://localhost:8000/docs

**تسجيل الدخول:**
- بيانات الدخول الافتراضية تُولَّد عشوائيًا عند أول تشغيل، وتُحدَّد من خلال متغيرات البيئة `INITIAL_ADMIN_USERNAME` و `INITIAL_ADMIN_PASSWORD` في ملف `.env`.
- لا توجد كلمات مرور افتراضية مدمجة في الكود.

### الطريقة 2: بدون Docker ⚙️

**تشغيل الباك إند:**
```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# عدّل .env بما يناسبك
python seed_db.py
uvicorn app.main:app --reload
```

**تشغيل الفرونت إند (نافذة جديدة):**
```bash
cd frontend
npm install
npm run dev
```

## 🔧 تعديلات أساسية قبل الاستخدام

### 1. تغيير كلمات المرور ⚠️
```bash
# في backend/seed_db.py
admin.set_password("YOUR_ADMIN_PASSWORD")
user.set_password("YOUR_USER_PASSWORD")
```

### 2. تغيير SECRET_KEY 🔑
```bash
# في backend/.env
SECRET_KEY=$(openssl rand -hex 32)
```

### 3. تغيير DATABASE_URL (لو لازم)
```bash
# في backend/.env
DATABASE_URL=postgresql://user:password@host:port/database
```
